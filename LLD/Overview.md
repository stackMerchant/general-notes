# LLD Overview

Topics:
- Design Principles
- Object-Oriented Concepts
- Design Patterns

Interview Assessment:
- Problem Analysis
- Class Design
- Code Quality
- Extensibility and Maintainability
- Communication

Delivery:
- Requirements, 5 min
- Entities and Relationships, 3 min
- Class Design, 10-15 min
- Implementation, 10 min
- Extensibility, 5 mins

Design Principles:
- KISS
- DRY
- YAGNI
- Separation of Concerns
- Law of Demeter, principle of least knowledge, reduce coupling, hide internal structure

Object-Oriented Design Principles (SOLID):
- SRP - Single Responsibility Principle - Keep classes focused on one responsibility
- OCP - Open/Closed Principle - Support future requirements without modifying existing code
- LSP - Liskov Substitution Principle - Prevent brittle hierarchies that break at runtime
- ISP - Interface Segregation Principle - Keep interfaces clean and focused
- DIP - Dependency Inversion Principle - Keep code flexible and testable through abstraction

OOP Concepts:
- Encapsulation
- Abstraction
- Polymorphism
- Inheritance

Design Patterns:
- Creational
    - Factory
    - Builder
    - Singleton
- Structural
    - Decorator, think encryption/validation/compression decorator wrapper on data r/w
    - Facade
- Behavioral
    - Strategy, replaces conditional logic with polymorphism
    - Observer
    - State Machine
